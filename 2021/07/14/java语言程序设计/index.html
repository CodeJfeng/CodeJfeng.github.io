第二章 Java的集合框架

引言

​ 在java中还引入了几种可以有效组织和处理数据的接口和类，这些接口和类组成了Java的集合框架。（Java Collection Framework）。

​ 一个集合就是储存数据的一个容器，一般将这些对象称为集合的元素。Java的集合构架支持三种类型：规则集（set）、线性表（list）和图（map），分别定义在Set、List与Map中。

注：在Java中，所有集合类都位于java.util包下，但支持多线程的集合类位于java.util.concurrent包下。

注：Java集合框架中所有的具体类都实现Cloneable和Serializable接口，所以他们的类都是可复制和可序化的。

规则集Set

​ Set接口扩展Collection接口，它没有引入新的方法和常量，只是规定它的实例不能包含相同的元素。。实现Set接口的具体类必须保证不能向它添加重复的元素。也就是说，在一个规则集中，存在元素e1，e2，使得e1.equal(e2)的返回值为true。

​ Set接口的三个具体类是HashSet、linkedHashSet和TreeSet。

散列集HashSet

​ HashSet类是实现接口的具体类。HashSet可以储存互不相同的任何元素。考虑到效率因素，添加到散列集的对象必须实现hashCode方法，适当地散列分布散列码。如Integer类中的haseCode方法返回的就是int值。Character类中的hashCode方法的返回值是字符的Unicode码。

例子说明：StringTokebizer方法提取单词，并把他们添加到规则集中。如果一个单词如“good”，添加多次，则只有一个会被存储，因为规则集中不允许有重复的元素。

​ 集合中的但粗没有按插入时的储存顺序，因为散列集中的元素没有特殊顺序。要给他们排序需要使用LinkedHashSet类。

链式散列集LinkedhashSet

​ 在HashSet中元素是没有顺序的，而在LinkedHashSet中，可以按元素插入集合的顺序进行提取，可以使用无参构造方法创建Linked Hash Set的对象。

例子说明：LinkedHashSet是一个规则集，所以不允许存储重复的元素。

如果不需要使元素按其插入的顺序进行存储，因该使用HashSet。它的效率比LinkedHashSet要高。

树形集TreeSet

SortedSet是Set的一个子接口，它保证规则集中的元素是有序的。TreeSet是实现SortedSet接口的具体类。只有当对象之间可以互相比较的时候，才可以将他们添加到树形集TreeSet中。有两种方法可以实现对象的比较：

  • 使用Comparable接口。由于添加到规则集内的对象是Comparable的实例，所以他们可以使用compareTo方法来进行比较。这种方法的定义通常称为自然顺序（natural order）
  • 如果该类不支持Comparable接口。或者在实现Comparable接口的类中不想使用CompareTo方法进行比较，可以给规则集的元素指定一个比较器。这种方法被称为比较器顺序。

​ 使用TreeSet方法，每次添加字符串的时候，都会给字符串重新排序。使用例子中的方法一般效率你较高，因为它只对字符串进行一次排序。

比较器接口Comparator

​ 当元素可能不是java.lang.Comparable的实例或者不能互相比较，可以定 义一个比较器来比较这些元素。要做到这点，创建一个java.util.Comparator接口的类、Comparator接口有两个方法compare和equals。

  • public int compare（Object element1 , Object element2）
  • public boolean equals(Object element)

Arrays类

本章小结

  • java集合构架支持三种集合：规则集、线性表和图。它们分别在Set、List和Map中定义。规则集set储存一组互不相同的元素。线性表list储存元素的有序集合。图map中储存一组对象，每个对象的键值不同
  • 有三种类型的规则集：散列集HashSet、链式散列集LinkedHashSet和树形集TreeSet。
  • 有两种类型的线性表：数组线性表ArrayList和链表LinkedList。
  • Vector类实现了List接口。在java 2中，Vector类与ArrayList基本一致，所不同的是它包含了访问和修改向量的同步方法。Stack类扩展了Vector类，并提供了集中管理栈的方法。
  • Collection接口表示储存在规则集或线性表中元素的集合。Map接口将键值映射为元素，键值类似于下标。在List中，下标为整数；在map中下标可以为任意一个值。
  • 有三种类型的图： HashMap、LinkedHashMap和TreeMap。对于查询值、插入映射和删除映射HashMap是最高效的。LinkedHashMap支持对图中元素的排序。在HashMap中是没有顺序的，但在LinkedHashMap中的元素可以按序访问。TreeSet对于按顺序遍历键值是高效率的，其中键值可以使用Comparable接口或Comparator接口排序。
  • Collections类提供对集合和图进行操作的静态方法，还提供了创建同步化集合。只读集合的静态方法。
  • Arrays类包含对数组的排序、查找、比较以及填充元素的静态方法。

第三章 多线程和国际化

引言

​ Java的重要功能之一就是天然的支持多线程。多线程是指在一个程序中允许同时运行多个任务的能力。在许多程序设计语言中，多线程都是通过调用依赖于系统的过程中或函数来实现的。本章将介绍线程的概念以及如何在Java中开发多线线程程序。

线程的概念

​ 一个线程（tread）指的是程序中完成一个任务的有始有终的执行流。使用java可以在一个程序中并发的运行多个线程，这些线程可以同时在多个处理器系统上运行。

​ 在单处理器系统中，多个线程分享CPU的时间，操作系统负责给它们调度和分配资。

​ 多线程可以使程序反应更快，交互性更强，执行效率更高。Java对多线程程序的开发和运行提供了很好的支持。

​ 在程序中还可以创建附加的线程以执行并发任务。每个新线程都是一个对象，它的类实现Runnable接口或者拓展实现Runnable接口。这种新对象称为可运行对象（runnable object）。

​ 既可以通过扩展Thread类，也可以通过Runnable接口来创建线程。Thread和Runnable都定义在包java.lang中。类Tread事实上实现了Runnable接口。

拓展Thread类创建线程

     Thread类包含创建线程的构造方法，以及控制线程的很多有用的方法。要创建并运行一个线程。首先定义Thread类的一个扩展类。自定义的线程类必须覆盖run( )方法，该方法告知系统如何执行线程。然后创建一个在线程上运行的对象。

问题： 编写一个程序，创建并运行三个线程：

  • 第一个线程打印字母a 100次
  • 第二个线程打印字母b 1000次
  • 第三个线程打印1~100的整数

例子说明：调用start( )方法启动一个线程，它调用了run( )方法。当run( ) 执行完毕，整个线程也就结束了。

实现runnable接口创建线程

​ 学习过Thread类扩展的用户类线程，创建和运行了一个线程。如果用户仅继承Thread类，这种做法很好；但是如果它要继承别的类，必须实现Runnable接口。

​ 若利用Runnable接口启动一个新线程，必须先创建一个实现Runnable接口的类的实例，然后使用Thread类的构造方法构造一个线程。

问题： 编写一个程序，创建并运行三个线程：

  • 第一个线程打印字母a 100次
  • 第二个线程打印字母b 1000次
  • 第三个线程打印1~100的整数

线程的状态

​ 线程有五种状态：新建、就绪、运行、阻塞或结束。

线程的优先级

​ java给每个线程指定一个优先级。可以用用setPriority方法提高或降低任何线程的优先级，还能用getPriority的方法得到线程的优先级。优先级的数字为1~10。

线程组

​ 一个线程组是线程的一个集合。有些程序包含相当多的具有类似功能的线程。为了方便，可以将他们放在一起作为一个整体进行操作。例如：可以同时挂起或唤醒同一组里的所有线程。

​ 以下列出使用线程组的主要步骤：

    1. 使用构造方法ThreadGroup来构造线程组：

ThreadGroup g = new ThreadGroup("thread group");

​ 上述语句创建了一个名为“Thread group”的线程组g。组名必须是唯一的字符串。

 2. 使用Thread构造方法，将一个线程放入到线程组中：

Thread t = new Thread( g , new ThreadClass( ) , "label for the thread");

​ 该语句创建了一个线程,并将它放入了线程组g中，可以把一个线程组添加到另一个线程组，构成一个树形结构，除了起始组以外，树中每一个线程组都一个父线程组。

 3. 使用activeCount( )确定有多少个线程组处于运行状态

System.out.println("The number is " + g.activeCount());

4. 每一个线程都属于一个线程组。默认情况下，一个新建的线程属于产生它的当前线程所在的组。可以用getThreadGroup( )方法判断线程属于哪一线程组。

线程之间的同步与协作

​ 如果一个共享资源被多个线程同时访问，可能会遭到破坏。

问题：编写一个程序冲突问题的程序。假设创建并启动了100个线程，每个线程都往同一个账户中添加一个便士编写一个程序冲突问题的程序。假设创建并启动了100个线程，每个线程都往同一个账户中添加一个便士。

例子说明：当所有线程同时访问一数据源时，就会出现诗句冲突的问题、通过调整线程的休眠时间可以显著的增加数据不稳定的可能性。这是多线程程序中普遍存在的问题，被称为资源冲突。如果一个类的对象在程序中没有引起资源冲突，则称这样的类为线程安全的。
